# 庙会摆摊核心系统

## 1. 概述

这个系统是《山野香火铺》当前可玩原型的核心玩法。
玩家在 3 天小节庆中经营单一摊口，通过选择摊口幡头、准备有限库存、接待来客、处理特殊来客和夜里布置，积累铜钱、口碑、缘分与热闹度。

## 2. 玩家体验目标

- 让玩家感受到“忙而不乱”的节庆经营节奏。
- 让每一次正确售卖、连单和特殊接待都有明确反馈和小成就感。
- 让中国山野庙会的烟火气通过来客、商品和结算自然体现出来。
- 保持轻经营取向，同时加入“白天经营，夜里布置”的连续经营层次。

## 3. 核心循环

1. 进入当天准备阶段。
2. 选择一个摊口幡头，决定当天偏重的商品与加成。
3. 庙会开市后逐个接待来客。
4. 判断来客需求，从 3 类商品中选择合适货品。
5. 通过卖对、连单、热闹度和特殊来客回礼累积资源。
6. 日结算后进入夜里布置，选择一个事件影响下一天。
7. 进入下一天，直到 3 天结束并触发结局。

## 4. 输入与输出

- 输入：
  左右键切换幡头或商品，空格/回车确认，鼠标可直接点击卡片，`R` 重开，`F` 全屏。
- 输出：
  铜钱、口碑、缘分、热闹度、当日结算结果、最终结局。
- 状态变化：
  当前模式在主菜单、准备阶段、摆摊阶段、结算阶段、夜间布置和终章之间切换。

## 5. 详细规则

### 阶段一：准备阶段

- 每天开始时，玩家从 3 个幡头中选择 1 个：
  - 山门清香：清香库存 +2，来客耐心更长。
  - 时鲜供果：供果库存 +2，正确卖出供果额外 +1 铜钱。
  - 福愿红绳：福牌库存 +2，正确卖出福牌额外 +1 缘分。
- 基础库存为每类商品 4 件，被选中的商品额外 +2。
- 每天还会有一个“今日地气”加成。
- 夜里布置带来的次日加成会在这一步一并生效。

### 阶段二：摆摊阶段

- 每位来客只明确需求 1 类商品：
  - 清香
  - 供果
  - 福牌
- 玩家为当前来客选择商品：
  - 卖对：获得商品基础铜钱，并增加口碑与缘分。
  - 卖错：只获得折损后的铜钱，口碑和热闹下降。
  - 等待超时：该来客离开，口碑和热闹额外下降。
- 当前来客结束后，下一位来客在短暂停顿后出现。
- 连续卖对会形成连单，部分节气会让连单额外提高收益。
- 热闹度越高，卖对时获得的铜钱加成越高。
- 特殊来客可能：
  - 给予额外即时奖励
  - 要求更快响应
  - 为下一天留下库存或规则加成

### 阶段三：日结算

- 当天所有来客处理完后，进入结算页面。
- 结算展示：
  - 成交数
  - 跑单数
  - 稀客数
  - 热闹度
  - 铜钱
  - 口碑
  - 缘分

### 阶段四：夜间布置

- 第 1 天和第 2 天结算后，玩家可以从 2 个夜间事件中选择 1 个。
- 夜间事件会消耗或提供即时资源，并为下一天追加经营修正。
- 常见修正包括：
  - 增加某类库存
  - 提高起始热闹
  - 增加来客耐心
  - 让特定商品卖对时得到额外收益

### 阶段五：终局

- 第 3 天结束后，根据总表现进入不同结局。

## 6. 边界情况

- 商品库存为 0 时，不能继续售卖该商品。
- 来客耐心耗尽后直接离开，不会等待补救。
- 玩家可以在摆摊中途按 `R` 重开整局。
- 即使连续卖错，也不会让游戏立刻失败，但会影响最终结局。
- 夜间事件如果资源不够，不能被选择。

## 7. 调参项

| 名称 | 含义 | 默认值 | 预计放在哪里 |
| --- | --- | --- | --- |
| 基础库存 | 每种商品的默认数量 | 4 | `src/gameplay/main.js`，后续迁移到 `assets/data/` |
| 幡头加成库存 | 被选中商品的额外库存 | 2 | `assets/data/festival-content.json` |
| 正确售卖口碑收益 | 卖对时增加的口碑 | 2 | `assets/data/festival-content.json` |
| 卖错惩罚 | 卖错时扣除的口碑 | -1 | `assets/data/festival-content.json` |
| 超时惩罚 | 来客流失时扣除的口碑 | -2 | `assets/data/festival-content.json` |
| 热闹度阶梯 | 热闹转化为铜钱加成的节奏 | 每 3 点一阶 | `assets/data/festival-content.json` |
| 夜间事件收益 | 夜里布置带给次日的修正 | 按事件而定 | `assets/data/festival-content.json` |
| 来客耐心 | 单位来客的等待时长 | 14 到 16 秒 | `assets/data/festival-content.json` |

## 8. 依赖关系

- 依赖 `design/gdd/game-concept.md` 中的整体题材、核心循环和第一版验证方式。
- 依赖网页画布渲染、输入处理和状态机逻辑。
- 依赖 `assets/data/festival-content.json` 提供节庆、商品、幡头、特殊来客与夜间事件。
- 后续若增加夜市阶段、更多节气或更长周目，需要继续扩展数据层和事件层。

## 9. 验收标准

1. 玩家能在 1 分钟内理解“选幡头 -> 接客 -> 售卖 -> 结算 -> 夜里布置”的完整循环。
2. 正确售卖、卖错和超时流失三种结果都能被明确区分。
3. 特殊来客和夜间事件能对次日经营产生可感知影响。
4. 3 天流程能稳定跑到结局，不出现状态卡死。
5. `render_game_to_text` 能准确反映当前模式、资源、库存、来客需求和夜间选项。
6. 截图中能明确看出主要按钮和当前操作目标。
